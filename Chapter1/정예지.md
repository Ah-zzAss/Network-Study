## Chapter1: 컴퓨터 네트워크와 인터넷

컴퓨터 네트워크의 가장자리와 코어에 대한 개요를 서술하는 챕터.

## 1. 인터넷이란 무엇인가?
인터넷은 다음과 같은 두자기로 서술할 수 있음

### 1.1 하드웨어와 소프트웨어 구성요소로 기술하는 인터넷

#### 호스트(host), 종단시스템 (end system)
다음의 모든 장치를 인터넷 용어로 부르는 단어
- 전통적인 인터넷 사물들 (데스크톱 PC, 리눅스 워크스테이션 등)
- 바전통적인 인터넷 사물들 (가전제품, TV)

#### 종단 시스템을 연결하는 것
이 종단 시스템은 다음두가지로 연결된다.
1. **통신 링크 (communication link)**
    - 동축케이블, 구리선, 광케이블 등 다양한 물리매체로 구성된다.
    - 다양한 전송률(링크 대역폭)을 이용하여 데이터를 전송한다.
    - 전송률은 bpd 단위를 사용한다.

2. **패킷 스위치/교환기 (packet switch)**
    - 입력 통신 링크의 하나로,  도착하는 패킷을 받아 출력 통신 링크의 하나로 그 패킷을 전달한다.
    - 최종 목적지 방향으로 패킷을 전달한다.
    - 종단 시스템 간의 데이터 교환에만 관여하고, 데이터의 시작과 끝인 애플리케이션에는 관심갖지 않는다.
    - **종류**
        1. 라우터 (router): 네트워크 코어에서사용된다
        2. 링크 계층 스위치 (link layer switch): 보통 접속 네트워크에서 사용된다.

> **한 종단 시스템이 다른 종단 시스템으로 데이터를 보낼 때**
> 1. 송신 종단 시스템은 데이터를 segment로 나눈다
> 2. 각 segment에 header를 붙여 **패킷**을 만든다.
> 3. 목적지 종단 시스템으로 네트워크를 통해 보내진다.
> 4. 목적지에서 다시 원래의 데이터로 조립된다.
> **경로(path)**: 패킷이 목적지에 도달하는 동안 거쳐온 일련의 통신링크, 패킷 스위치

### 1.2 서비스 측면에서 본 인터넷

애플리케이션에 서비스를 제공하는 인프라스트럭처로서 인터넷을 기술한다.

- 인터넷은 많은 애플리케이션을 포함한다.
- 이러한 애플리케이션은 서로 데이터를 교환하는 많은 종단시스템을 포함하기 때문에 분산 애플리케이션이라고 부른다.

- **소켓 인터페이스**: 특정 목적지 프로그램으로 데이터를 전달하도록 요구할지 명시

### 1.3 프로토콜

둘 이상의 통신 개체 간에 교환되는 메세지 포맷, 순서 뿐만 아니라 메세지의 송수신과 다른 이벤트에 따른 행동들

- 모든 활동은 프로토콜이 제어한다.
- 물리적으로 연결된 두 컴퓨터의 네트워크 접속카드에 하드웨어로 구현되어 있음

## 3. 네트워크 코어

인터넷의 종단 시스템을 연결하는 패킷 스위치아 링크의 그물망
링크와 스위치를 통해 데이터를 이동시키는 방식: 회선 교환(3.1), 패킷 교환(3.2)

### 3.1 패킷 교환

#### 패킷
종단 시스템이 보내는 메세지를 작은 데이터 덩어리로 분할한 것

- 각 패킷은 통신 링크, 패킷 스위치를 거치게 된다
- 저장 후 전달 전송 방식을 이용한다.

#### 패킷 스위치

- 여러 개의 접속된 링크를 갖고 있다.
- 각 링크에 대해 패킷 스위치는 출력 buffer, 출력 queue를 갖고 있다.

- **패킷을 전달하는 방법 : 저장 후 전달**
    - 패킷 스위치가 출력 링크로 첫 비트를 전송하기 전에 전체 패킷을 받아야함
    - 수신한 비트를 먼저 저장한다
    - 라우터가 패킷의 모든 비트를 수신한 후에만 출력 링크로 그 패킷을 전송

- **큐잉 지연**: 패링크에서 다른 패킷을 전송하고 있다면, 도착하는 패킷은 출력 버퍼에서 대기
- **패킷 손실**: 버퍼의 크기가 유한하기 때문에 꽉 차있는 경우 도착하는 패킷 혹은 이미 큐에 대기 중인 패킷을 폐기

> #### 패킷 교환
> - **패킷 교환 반대자:** 가변적, 예측할 수 없는 종단 간의 지연때문에 실시간 서비스에 적당하지 않다.
> - **패킷 교환 찬성자:**
>   - (1) 회선 교환보다 전송 용량의 공유에서 더 효율적
>   - (2) 패킷 교환이 더 간단하고 효율적, 회선교환보다 구현 비용이 적다.

#### 포워딩 테이블

목적지 주소를 라우터의 출력 링크로 매핑

- 각 라우터는 포워딩 테이블을 이용하여 패킷을 어느 링크로 전송할지 결정한다,
- 패킷이 라우터에 도착하면 올바른 출력 링크를 찾기 위해 주소를 조사한다. 그 목적지 주소를 이용하여 포워딩 테이블을 검색 후 출력 링크로 전달

#### 라우팅 프로토콜

자동으로 포워딩 테이블을 설정하는데 이용되는 프로토콜

### 3.2 회선 교환

종단 시스템 간에 통신을 제공하기 위해 경로상 필요한 자원은 통신 session 동안에 확보/예약된다.  
링크 내 한 회선은 주파수 분할 다중화 혹은 시분할 다중화로 구현된다.

### 주파수 분할 다중화
링크를 통해 설정된 연결은 그 링크의 주파수 스펙트럼을 공유한다.  
여기서 링크가 연결되는 동안 각 연결에 대해 주파수 대역을 고정 제공한다.

### 시분할 다중화
시간을 일정 주기의 프레임으로 구분하고, 각 프레임은 고정된 수의 시간 슬롯으로 나뉜다.  
네트워크는 모든 프레임에서 시간 슬롯 1개를 그 연결에 할당한다.

### 3.3 네트워크의 네트워크

#### ISP
패킷 스위치와 통신 링크로 이루어진 네트워크

- ISP는 종단 시스템에게 다양한 네트워크 접속을 제공
- 웹사이트와 비디오 서버를 직접 연결하도록 CP(content provider) 에게 인터넷 접속을 제공
- 인터넷은 종단 시스템을 서로 연결하는 것이므로 종단 시스템에게 접속을 허용하는 ISP도 서로 연결 되어야함

#### 네트워크 구조
접속 ISP들은 서로 연결되어야만 한다. 이를 위해 네트워크의 네트워크가 등장하게 되었다.

- **네트워크 구조1:** 모든 접속 ISP를 하나의 글로벌 통과 ISP로 연결
    - 접속 ISP가 글로벌 ISP에게 비용을 지불
    - 비용을 지불하는 ISP는 고객, 제공하는 ISP는 제공자

- **네트워크 구조2:** 수십만개의 접속 ISP와 글로벌 ISP로 구성
    - 상위 계층에 글로벌 통과 서비스 제공자가 있고 하위층에 접속 ISP가 있는 형태의 2계층 구조
    - 특정 지역에 있는 접속 ISP들이 연결하는 지역 ISP가 있고, 각 지역 ISP들은 1계층 ISP들과 연결

**네트워크 구조3:** 각 도시 ISP -> 각 지방 ISP -> 각 국가 ISP-> 최종적으로 1계층 ISP에 연결되는 다중 계층 구조

현재의 인터넷과 더 유사한 네트워크를 구축하기 위해서는 다음을 네트워크 구조3에 포함시켜야함. 
- PoP (point of reference) : 제공자의 네트워크 내에 있는 하나 이상의 라우터 그룹
- 멀티홈: 둘 이상의 제공자 ISP에 연결하도록 선택하는 것
- 피어링: 송수신되는 모든 트래픽을 상위 계층 ISP를 통하지 않고 직접 송수신할 수 있도록 서로의 네트워크를 연결하는 것
- IXP: 다중의 ISP이 서로 피어링할 수 있는 만남의 장소
