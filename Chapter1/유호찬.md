# 인터넷이란 무엇인가?

구성요소로 본 인터넷

- 다양한 컴퓨팅 장치들을 호스트 or 종단 시스템으로 불린다.
- 각각의 종단 시스템은 통신 링크와 패킷 스위치로 연결된다.
- 다양한 ISP가 종단 시스템에게 인터넷 접속을 제공한다.
- 인터넷 주요 프로토콜을 TCP/IP라고 하며 이외에도 다양한 프로토콜이 존재한다.

서비스 측면에서 본 인터넷

- 인터넷은 애플리케이션에 서비스를 제공하는 인프라스트럭처
- 애플리케이션은 서로 데이터를 교환하는 많은 종단 시스템을 포함하기 때문에 분산 애플리케이션(distributed application)이라고 부른다.
- 각 종단 시스템들은 서로 데이터를 전달하기 위해 규칙을 명시하는 소켓 인터페이스(socket interface)를 제공한다.

프로토콜이란 무엇인가?

- 프로토콜 : 둘 이상의 통신 개체 간에 교환되는 메시지 포맷과 순서뿐만 아니라, 메시지의 송수신과 다른 이벤트에 따른 행동들을 정의
- 어떠한 일을 수행하려면 둘 이상의 통신 개체(entity)가 함께 이신하는 프로토콜이 필요

# 네트워크의 가장자리(edge)

종단 시스템(호스트) : 데스크톱, 서버, 노트북 등등
종단 시스템(호스트)는 클라이언트와 서버로 구분된다.

접속 네트워크

- 가정 접속 : DSL, 케이블, FTTH, 5G 고정 무선
- 기업(가정) 접속 : 이더넷, 와이파이 (초기에는 기업환경에서 구축되었지만 최근에 홈 네트워크의 공통요소이다)

물리 매체

1. 꼬임쌍선 : 가장 저렴하고 많이 이용하는 전송 매체
2. 동축케이블 : 2개의 구리선, 꼬임쌍선보다 더 높은 데이터 전송률, 케이블 TV 시스템에 흔히 사용
3. 섬유 : 빛의 파동을 전하는 전송 매체, 고가이므로 가정 근거리 전송보다 해저 링크 같은 곳에서 사용
4. 지상 라디오 채널 : 전자기 스펙트럼으로 신호 전달, 환경과 거리에 따라 많은 영향을 받음
5. 위성 라디오 채널 : 정지 위성과 저궤도 위성, 정지 위성은 신호 전파 지연을 일으키지만 케이블 기반 인터넷 접속을 할 수 없는 지역에서 주로 이용

# 네트워크 코어

패킷 교환

- 패킷 : 메세지를 작은 데이터 덩어리로 분리한 것, 통신 링크와 패킷 스위치를 거침
- 저장 후 전달(store-and-forward transmission) : 패킷의 첫 비트를 전송하기 전 전체 패킷을 받아야함
- 종단 간 지연 = N*(L/R)
- 링크가 다른 패킷을 전송하고 있다면 출력 버퍼에서 **큐잉 지연**이 일어나고 버퍼가 꽉 차 있을 경우 **패킷 손실**이 발생한다.
- 라우터는 **포워딩 테이블**을 통해 패킷을 올바른 링크로 전달한다
- **포워딩 테이블**은 **라우팅 프로토콜**에 의해 결정된다

회선 교환

- 패킷과 다르게 스위치들이 해당 연결 상태를 유지해야 하는 연결이 회선 ex) 통화
- 보장된 전송률로 데이터를 보낼 수 있다.
- 주파수 분할 다중화(FDM) : 연결되는 동안 주파수 대역을 고정 제공하고 공유한다.
- 시분할 다중화(TDM) : 시간을 프레임과 시간슬롯으로 나누어 연결한다.

패킷 교환 VS 회선 교환

- 회선 교환 : 보장된 연결을 지원한다.
- 패킷 교환 : 전송 용량, 구현 비용 등에서 회선 교환 보다 효율적이다.
- 회선 교환 전화망이 패킷 교환으로 전환되면서 패킷 교환이 널리 사용되고 있다.

네트워크의 네트워크

- 오늘날의 인터넷은 매우 복잡하다.
- 12개의 1계층 ISP와 수십만 개의 하위 계층 ISP로 구성
- 하위 계층 ISP는 상위 계층 ISP들과 연결하고 상위 계층 ISP들은 서로 연결한다.

# 패킷 교환 네트워크에서의 지연, 손실과 처리율

패킷이 한 호스트에서 시작하여 라우터들을 통과하고 다른 호스트에 도착하는 과정에서 다양한 지연을 겪게된다.

1. 처리지연(processing delay) : 패킷 헤더를 조사하고 그 패킷을 어디로 보낼지를 결정하는 시간
2. 큐잉지연(queuing delay) : 큐에서 링크로 전송되기를 기다리는 시간. 다른 패킷이 많다면 길어짐
3. 전송지연(transmission delay) : 패킷을 내보내는 시간. L(패킷)/R(전송률)
4. 전파지연(propagation delay) : 패킷이 실제로 이동하는 시간. 거리와 물리 매체에 따라 정해진다.
5. 전체노드지연 : 4가지 지연을 더한 것

트래픽 강도 : La/R, 비트가 큐에 도착하는 정도와 비트가 큐에서 전송되는 비율

- 트래픽 강도가 1을 초과하게 된다면 큐는 끝없이 증가하며 큐잉 지연이 무한대에 도달한다.
- La/R < 1로 설계해야 하지만 1에 가까울수록 평균 큐잉 지연이 급속도로 높아진다.
- 큐가 꽉 차도록 큐잉 지연이 일어나게 된다면 패킷을 버리는 패킷 손실이 일어난다.

종단 간 지연

- N-1개의 라우터가 있다고 할 때
- 네트워크가 혼잡하지 않으며
- $d_{\text{end-to-end}} = N(d_{\text{proc}} + d_{\text{trans}} + d_{\text{prop}})$

컴퓨터 네트워크에서의 처리율

- 처리율이 다른 여러 링크로 패킷이 전달된다면 가장 낮은 처리율이 종단 간 처리율이다.
- 공통 링크의 경우 간섭 트래픽으로 인하여 속도가 더 느려질 수 있다.

# 프로토콜 계층과 서비스 모델

계층 구조

1. 해당 계층에서 어떤 동작을 취하고
2. 그 계층 바로 아래 계층 서비스를 이용함으로써 서비스를 제공한다.

프로토콜 스택

- 다양한 계층의 프로토콜을 모두 합하여 프로토콜 스택이라고 한다.
- 물리, 링크, 네트워크, 트랜스포트, 애플리케이션으로 5개 계층으로 구성된다.

1. 애플리케이션 계층
    1. 각 종단 시스템에서 애플리케이션 끼리 정보 패킷을 교환
    2. 정보 패킷 : 메시지(message)
2. 트랜스포트 계층
    1. 클라이언트와 서버 간에 애플리케이션 계층 메시지를 전송
    2. TCP, UDP 두 가지 트랜스포트 프로토콜 사용
    3. TCP : 연결지향형 서비스, 계층 메시지 전달 보장
    4. UDP : 비연결형 서비스, 계층 메시지 전달을 보장하지 않는다. 
    5. 패킷 : 세그먼트(segment)
3. 네트워크 계층
    1. 한 호스트에서 다른 호스트로 데이터그램을 라우팅
    2. 트랜스포트 계층에서 세그먼트와 목적지 주소를 네트워크 계층에 전달
    3. 목적지 트랜스포트 계층으로 세그먼트를 운반
    4. IP 프로토콜과 라우팅 프로토콜을 가지고 있다.
    5. 패킷 : 데이터그램(datagram)
4. 링크 계층
    1. 네트워크 전체적인 라우팅을 말한다면 링크계층에서는 한 노드에서 다른 노드를 이동하는 서비스를 말한다.
    2. 패킷 : 프레임(frame)
5. 물리 계층
    1. 프레임 내부의 각 비트를 다음 노드로 이동하는 서비스
    2. 링크의 실제 전송 매체(꼬임쌍선, 광케이블)에 의존한다.

캡슐화

- 호스트는 5 계층을 모두 구현하지만 라우터는 1~3(물리, 링크, 네트워크), 링크 계층 스위치(물리, 링크)만 구현한다.
- 캡슐화
    - 각 계층마다 애플리케이션 메세지에 헤더를 붙여서 다음 계층으로 전달한다.
    - 각 계층의 패킷은 헤더 필드와 페이로드 필드가 존재한다.
    - 페이로드는 일반적으로 상위 계층의 패킷이다.
    - 각 계층을 거치며 캡슐화하고 송신 호스트에 도착하면 반대 과정을 진행한다.
